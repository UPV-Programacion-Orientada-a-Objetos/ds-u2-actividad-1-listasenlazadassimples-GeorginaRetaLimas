\doxysubsection{Detailed Description}
Administrador central de todos los sensores del sistema. 

TAREAS\+:
\begin{DoxyEnumerate}
\item Mantener una lista de todos los sensores activos
\item Permitir agregar nuevos sensores
\item Buscar sensores por ID
\item Procesar todos los sensores con iteración
\item Liberar memoria al destruirse
\end{DoxyEnumerate}

PATRÓN DE DISEÑO\+: Este es un ejemplo del patrón "{}\+Gestor"{} o "{}\+Manager"{}\+:
\begin{DoxyItemize}
\item Centraliza la gestión de recursos (sensores)
\item Proporciona una interfaz limpia para operaciones comunes 
\end{DoxyItemize}

Destructor -\/ Libera TODOS los sensores y sus listas internas. 

PROCESO CRÍ\+TICO DE LIBERACIÓN EN CASCADA\+:

Para cada sensor en la lista\+:
\begin{DoxyEnumerate}
\item Obtenemos el puntero Sensor\+Base\texorpdfstring{$\ast$}{*}
\item delete sensor llama a\+: a) \texorpdfstring{$\sim$}{\string~}\+Sensor\+Temperatura() o \texorpdfstring{$\sim$}{\string~}\+Sensor\+Presion()
\begin{DoxyItemize}
\item Aquí se imprime el mensaje de liberación
\item Se destruye \textquotesingle{}historial\textquotesingle{} (\doxylink{classListaSensor}{Lista\+Sensor})
\item \texorpdfstring{$\sim$}{\string~}\+Lista\+Sensor() libera TODOS los nodos internos b) \texorpdfstring{$\sim$}{\string~}\+Sensor\+Base()
\end{DoxyItemize}
\item Liberamos el \doxylink{structNodoSensor}{Nodo\+Sensor}
\end{DoxyEnumerate}

RAZÓN del orden\+:
\begin{DoxyItemize}
\item Primero delete sensor (libera el sensor y su lista interna)
\item Después delete nodo\+Actual (libera el nodo de gestión)
\end{DoxyItemize}

Si no hiciéramos esto = FUGA MASIVA DE MEMORIA\+:
\begin{DoxyItemize}
\item Los sensores quedarían huérfanos
\item Sus listas internas nunca se liberarían
\item Todos los nodos de lecturas quedarían ocupando RAM 
\end{DoxyItemize}
